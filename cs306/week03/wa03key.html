Exercise 7 on page 149

Pseudocode for a recursive algorithm for generating all 2n bit strings of length n:

Algorithm BitstringsRec(n)
  // Generates recursively all the bit strings of a given length
  // Input: A positive integer n
  // Output: All bit strings of length n as contents of global array B[0..n-1]
  if n = 0
     print(B)
  else
     B[n - 1] ← 0; BitstringsRec(n - 1)
     B[n - 1] ← 1; BitstringsRec(n - 1)

        (b (eval-minibuffer "Enter Multiplier as Lisp Expression: ")))

(defun rpm (a b)
  (let ((rim 0)
        (sum 0))
    (while (>= b 1)
      (setq rim (% b 2)
            b (/ b 2))
      (cond ((= 1 rim)
             (setq sum (+ sum a))
             (princ (format "%d\n" sum))))
      (setq a (* a 2)))
    sum))

Exercise 1 on page 166

a. Since the algorithm uses the formula gcd(m, n) = gcd(n, m mod n), the size of the new pair will be m mod n. Hence it can be any integer between 0 and n - 1. Thus, the size n can decrease by any number between 1 and n.

b. Two consecutive iterations of Euclid's algorithm are performed according to the following formulas:

gcd(m, n) = gcd(n, r) = gcd(r, n mod r) where r = m mod n.

Need to show that \(n\ \mbox{mod}\ r \le n/2\). Consider two cases: \(r \le n/2\) and \(n/2 < r < n\). If \(r \le n/2\), then

\(n\ \mbox{mod}\ r < r \le n/2.\)

If \(n/2 < r < n\), then

\(n\ \mbox{mod}\ r = n - r \le n/2,\)

too.

Exercise 12 on page 167

Here is a decrease-and-conquer algorithm for solving this problem. Repeat the following until the problem is solved: Find the largest pancake that is out of order. (If there is none, the problem is solved.) If it is not on the top of the stack, slide the flipper under it and flip to put the largest pancake on the top. Slide the flipper under the first-from-the-bottom pancake that is not in its proper place and flip to increase the number of pancakes in their proper place at least by one.
